Configuration drift detection

Know before you deploy.

Drift compares your environment configurations — staging, production, and everything in between — so you catch mismatches before they become incidents.

Start drift scan View on GitHub
drift compare staging..production 
# Scan two environment configs in seconds
$ drift compare --from staging --to production

Env: staging -> production

- AWS_REGION=us-east-1
+ AWS_REGION=us-west-2

- LOG_LEVEL=debug
+ LOG_LEVEL=info

- FEATURE_FLAGS=replay,canary
+ FEATURE_FLAGS=replay

3 drifted keys found • 1 missing in production

Everything you need to stay in sync

Drift fits into your existing workflow. No dashboards to learn, no agents to manage — just fast, reliable configuration diffing.

Compare any environments

Diff staging against production, preview deploys, or any two state files. Drift reads JSON, YAML, .env files, Terraform outputs, and cloud parameter stores.

CI guardrails

Add one line to your pipeline and block merges when drift is detected. Exit codes make it trivial to integrate with GitHub Actions, GitLab CI, or CircleCI.

Fast local CLI

Native binaries for macOS, Linux, and Windows. Most scans complete in milliseconds, so you can run Drift on every commit without slowing down.

Add it to your pipeline in minutes

Drift is designed to be composed with the tools you already use. Define sources, run a comparison, and let your CI handle the rest.

drift.yaml example config 
version: 1

environments:
  staging:
    source: ssm
    path: /app/staging
  production:
    source: ssm
    path: /app/production

rules:
  - name: required_keys_match
    severity: error
  - name: values_unchanged
    severity: warning
    ignore:
      - LOG_LEVEL
