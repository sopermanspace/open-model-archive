Know before you deploy.

Drift is the CLI that detects configuration drift between your staging, QA, and production environments—guaranteeing consistency every time.

Start drift scan

Why Drift? Never Guess, Always Verify.

Deployment issues often stem from subtle configuration differences (the "drift"). Drift identifies and quantifies these discrepancies across your entire stack.

📸 Snapshot Tracking

Takes immutable snapshots of environment states—files, databases, services. Every change is logged, making debugging trivial and rollbacks precise.

🔥 Variance Reporting

Generates crystal-clear, actionable reports detailing exactly *what* drifted (e.g., a service version update or an env variable change) and where.

✅ Audit Trail Compliance

Built for regulated industries. Provides cryptographically verifiable logs showing who changed what, when, and why, satisfying the toughest compliance requirements.

Usage Demo

Run this command in your CI/CD pipeline to detect all unauthorized changes between the `staging` and `production` branches:


$ drift scan --environments staging production 

# Successful run example:
✔ Drift Scan Complete.
