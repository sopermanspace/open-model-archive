Build a **complete, playable, visually stunning** endless lane-crossing arcade game called **Voxel Hop** — inspired by classic hopper games (like Crossy Road's *mechanics*, but **not** its branding, characters, or art).

## Visual direction (must feel premium)

- **2.5D isometric voxel aesthetic** — chunky blocks, soft shadows, depth via offset layering
- **Vibrant sky gradient** (dawn palette: coral → lavender → deep blue) with subtle parallax clouds
- **Juicy motion**: hop squash/stretch, screen shake on death, coin sparkle particles, smooth camera lerp
- **Polished UI**: rounded score pill, coin counter with icon, clean sans-serif type (system fonts only)
- The game should look **stunning in a screenshot** — not a wireframe prototype

## Required game systems (all must work)

1. **Title screen** — animated logo "VOXEL HOP", floating character, "Press Space or Tap to Play"
2. **Grid movement** — discrete hops on a lane grid: up/down/left/right (one tile per hop, no diagonal)
3. **Lane types** (procedurally generated ahead of the player):
   - **Grass** — safe resting lanes
   - **Road** — cars and trucks moving left/right at varied speeds; getting hit = death
   - **River** — water lane with floating logs/rafts; player must be on a raft or drowns
   - **Railroad** — periodic fast train crossing; instant death if on tracks when train passes
4. **Coins** — scattered on safe tiles; collecting adds bonus score with a brief sparkle effect
5. **Scoring** — +1 per lane advanced forward; display current score and best score (`localStorage`)
6. **Camera** — follows player upward as they progress; despawn lanes far behind
7. **Infinite generation** — spawn new lane rows ahead with weighted random lane types; difficulty ramps (faster vehicles over time)
8. **Death + restart** — collision triggers particle burst + brief freeze, then game-over overlay with score and "Play Again"
9. **Controls** — Arrow keys + WASD; touch/swipe on mobile (up/down/left/right swipe zones or tap edges)
10. **Game states** — `title` → `playing` → `gameover` with clean transitions

## Technical requirements

1. Output **only** a single fenced `html` code block containing the full page. No explanation outside the code block.
2. Self-contained: embedded CSS and JavaScript only. **No external libraries, fonts, images, or CDN links.**
3. Use **HTML5 Canvas** for rendering (recommended for performance) or DOM — must run at smooth 60fps on a laptop.
4. Draw all graphics procedurally (rectangles, paths, gradients) — no sprite sheets or external assets.
5. Semantic HTML wrapper; canvas centered and responsive (scale to fit viewport, maintain aspect).
6. Do **not** use Crossy Road, Flappy Bird, Snake, Tetris, Pac-Man, or Mario names/sprites.
7. Do **not** ship a broken stub — every system above must be implemented and playable on first load.

The result should feel like a premium mobile arcade hit — charming, colorful, and immediately fun.
