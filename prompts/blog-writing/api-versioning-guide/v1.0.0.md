Write a **tutorial-style blog post** (700–900 words) titled:

**"How to Version REST APIs Without Breaking Your Mobile Clients"**

## Requirements

- Audience: backend developers new to API versioning
- Tone: friendly, practical, step-by-step — **not** the same authoritative essay style as a post-mortem or incident write-up
- Structure: short intro, 3 numbered sections with `##` headings, at least one inline code example (HTTP path or header), a "common mistakes" subsection, closing checklist
- Cover URL versioning vs header versioning with a clear recommendation
- Use second person ("you") sparingly but naturally
- Flowing prose — avoid bullet-point walls
- End with a one-sentence takeaway

Output plain text markdown. No preamble.
