Explain **`git rebase`** to a home cook who has never used version control.

## Rules

1. Stay technically accurate — commits, branch, replay, conflicts must be correct
2. Use cooking/kitchen metaphors throughout (prep bowls, sauce reduction, mise en place)
3. Contrast rebase with merge using one food analogy
4. Length: 250–400 words
5. The cook should "respond" once at the end with a single sentence in quotes

Output plain text only. No title, no markdown headers.
