About

Open Model Archive is a public, reproducible record of what AI models generate when given the same real-world tasks.

What this is

Not a leaderboard. Not a single score. A transparent archive where developers can inspect exact outputs, artifacts, timing, and token usage side by side.

How it works

  1. Versioned prompts live in the repository.
  2. The build pipeline executes each task against registered models.
  3. Outputs, metadata, and screenshots are committed as public artifacts.
  4. A static site is generated for effortless comparison.

Any model, any provider

The archive is model-agnostic. Contributors run whatever they have access to — there is no required model stack. Wire up a YAML config, authenticate locally, execute tasks, and publish the artifacts.

  • Ollama (ollama) — any local or cloud-backed Ollama model
  • Provider CLI (cli) — any model behind an installed CLI
  • OpenAI (openai) — GPT and compatible APIs via OPENAI_API_KEY
  • Anthropic (anthropic) — Claude via ANTHROPIC_API_KEY
  • OpenRouter (openrouter) — any model on OpenRouter
  • Together AI (together) — hosted models via TOGETHER_API_KEY
  • Fireworks (fireworks) — hosted models via FIREWORKS_API_KEY
  • Sarvam AI (sarvam) — via SARVAM_API_KEY
  • Codex (codex) — OpenAI Codex-compatible endpoints

Disabled templates ship in models/. Enable locally, never commit API keys. The live site currently shows Gemma 4, Kimi K2.7 Code, and Gemini 3.1 Pro as one published comparison set — your PR can add any other model.

Task categories

Most categories include multiple tasks in different styles — dark SaaS vs editorial restaurant pages, tutorial vs opinion blogs, icon sets vs diagrams — so comparisons show range, not a single data point.

  • Website generation — developer SaaS, indie game waitlist, editorial restaurant (3 tasks)
  • Code generation — rate limiter, LRU cache, slugify utility (3 tasks)
  • Blog writing — incident essay, API tutorial, monorepo opinion (3 tasks)
  • SVG generation — weather icons, app logo, onboarding diagram (3 tasks)
  • Web grounding — product docs Q&A, remote policy Q&A (2 tasks)
  • Vision understanding — full visual analysis, typography audit (2 tasks)
  • Design critique — desktop review, mobile-first critique (2 tasks)
  • Humor testing — golden retriever, git-as-cooking (2 tasks)
  • Creative fiction — press release, product review (2 tasks)
  • Math — elementary, college, river-crossing logic (3 tasks)
