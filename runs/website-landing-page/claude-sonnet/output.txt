  <section class="code-section" id="code-example">
    <div class="code-columns">
      <div>
        <h2>One command, wired into CI</h2>
        <p>Drift is designed to live in your pipeline, not in a browser tab. Point it at two environments and let it fail the build when something looks wrong.</p>
        <ul>
          <li>Runs entirely from a single static binary — no daemon, no agent installed in your cluster.</li>
          <li>Exit codes map to severity, so CI gating takes one line.</li>
          <li>Supports ignore rules for expected differences like hostnames or replica counts.</li>
        </ul>
      </div>
      <pre class="code-block mono"><span class="c-com"># .github/workflows/deploy.yml</span>
- name: <span class="c-str">Check for config drift</span>
  run: |
    drift check \
      <span class="c-flag">--from</span> env:staging \
      <span class="c-flag">--to</span> env:production \
      <span class="c-flag">--fail-on</span> <span class="c-key">error</span> \
      <span class="c-flag">--ignore-file</span> .driftignore

<span class="c-com"># .driftignore</span>
<span class="c-key">HOSTNAME</span>
<span class="c-key">REPLICA_COUNT</span>
<span class="c-key">DEPLOY_TIMESTAMP_*</span></pre>
      </div>
