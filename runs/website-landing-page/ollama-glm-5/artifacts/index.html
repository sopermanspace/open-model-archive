Catch config drift before it breaks prod.

Drift compares your environments and surfaces inconsistencies in secrets, env vars, and config files. Know before you deploy.

Start drift scan

Features

Stop trusting your configs blindly.

Diff detection

Scan any two environments and get a structured report of every configuration mismatch. Works with .env, YAML, JSON, and TOML.

Secrets scanning

Automatically detect exposed secrets and missing credentials across your environments before they cause incidents.

CI integration

Fail your pipeline on drift. Drop-in GitHub Action, GitLab CI template, and standalone binary for any workflow.

Example

One command, full visibility.

terminal 
# Compare staging and production
$ drift scan --env staging --compare production

Scanning [.env, config.yaml, secrets/]

- DATABASE_URL mismatch
    staging:    postgres://db.internal:5432/app
    production: postgres://db.prod:5432/app

- REDIS_MAX_CONNS missing in production
    staging:    100
    production: (undefined)

+ 2 secrets detected in staging only
+ 1 deprecated key in production

Summary: 3 drift items found. Run drift report for details.
