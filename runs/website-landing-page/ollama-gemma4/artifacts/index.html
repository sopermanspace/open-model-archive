How it works: Scanning a Cloud Region

Run the command in your CI/CD pipeline to detect discrepancies between 'target' (Prod) and 'source' (Staging).


$ drift --project infra-backend \
  --environments staging,production \
  --targets terraform/main.tfstate \
  --output json > drift_report.json

Example output snippet:
Resource 'aws:s3/bucket-dev' exists in Staging, but not found in Production target state.
